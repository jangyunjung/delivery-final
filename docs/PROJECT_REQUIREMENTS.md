# 기말 프로젝트 요구사항 요약

## 목표
배달앱을 만들고 GitHub → Vercel로 배포한다.
평가 핵심은 실제 작동, DB 구조 이해, 버그 원인 설명이다.

## 필수 기능
1. 회원가입 / 로그인 / 로그아웃
2. 식당 · 메뉴 목록 보기
3. 장바구니 담기
4. 주문하기, DB 저장
5. 내 주문 내역 보기

## 제출물
1. 영상
- 공개 URL에서 회원가입 → 로그인 → 식당/메뉴 → 장바구니 → 주문 → 주문내역 → 로그아웃 시연
- DB 구조 설명
- 막혔던 버그 1개의 원인과 해결
- 회고

2. 문서
- DB 구조도
- 막혔던 지점 3개와 해결
- AI로 안 풀린 것 / 한계

3. 링크
- Vercel 공개 URL
- GitHub 저장소 주소

## 확정 DB 구조
- users
- restaurants
- menu_items
- orders
- order_items

## DB를 이렇게 나누는 이유

restaurants와 menu_items는 식당 1개가 여러 메뉴를 가지기 때문에 나눈다.

orders와 order_items는 주문 1건에 여러 메뉴가 들어갈 수 있기 때문에 나눈다.

order_items에는 주문 당시 메뉴명과 가격을 snapshot으로 저장한다.
이렇게 하면 나중에 메뉴 가격이 바뀌어도 과거 주문 내역의 금액이 바뀌지 않는다.

## 구현 우선순위
1. 필수 기능 5개 정상 작동
2. Vercel 공개 URL 배포
3. 주문 한 바퀴 성공
4. DB 구조 설명 가능
5. 버그 1개 원인-해결 기록
6. 디자인은 마지막에 최소 정돈
